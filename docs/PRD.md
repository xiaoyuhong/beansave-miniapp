# BeanSave 咖啡余量小程序 · 产品需求文档（PRD）

> 版本：1.1.0
> 更新日期：2026-05-12
> 产品类型：微信小程序（Taro 4 + React 18 + TypeScript + NutUI + Vite）

---

## 1. 项目背景

BeanSave 咖啡（大学城店）位于上海市松江区，每天下午 3 点后有 20~30 杯未售出的精品咖啡。当前通过微信群半价售卖，效率低且无法规模化。

**目标**：开发一款微信小程序，实现每天 15:00 自动上架余量咖啡，用户半价抢购 → 到店扫码核销。

## 3. 用户角色

| 角色                  | 说明                                    |
| --------------------- | --------------------------------------- |
| C端用户（role: user） | 附近学生、上班族，购买余量咖啡          |
| 商家（role: admin）   | 店长/店员，发布余量、核销订单、查看统计 |

**角色判断**：后端登录接口返回 `role` 字段（`user` / `admin`），前端根据 role 动态切换 TabBar 和可访问页面。

### 商家如何获得 admin 角色

提供两种方式，可按实际情况选择：

| 方式                            | 说明                                                                       | 适用场景                                |
| ------------------------------- | -------------------------------------------------------------------------- | --------------------------------------- |
| **方式一：后端手动配置 openid** | 店长首次登录后，由开发者在数据库中将其 openid 对应的 role 字段改为 `admin` | 店铺规模小、商家固定，推荐 MVP 阶段使用 |
| **方式二：邀请码机制**          | 后端生成一次性邀请码，店长在小程序内输入邀请码后自动升级为 `admin`         | 多店铺扩展、商家自助开通，推荐后期使用  |

> MVP 阶段使用**方式一**，开发者直接在数据库执行：
>
> ```sql
> UPDATE users SET role = 'admin' WHERE openid = 'xxx';
> ```
>
> 后期如需支持多店铺或自助开通，再接入邀请码机制。

## 4. 登录方式

- 微信一键授权（获取头像 + 昵称）
- 登录后引导获取手机号（微信授权，非强制）
- 后端记录 openid，返回 token + role

## 8. 附加功能说明

### 距离检测（P2）

使用**腾讯位置服务**实现，需提前申请 Key。

**接入步骤：**

1. 前往 [腾讯位置服务控制台](https://lbs.qq.com/) 注册并创建应用，获取 `WebServiceAPI Key`
2. 在微信小程序管理后台 → 开发设置 → 服务器域名，添加 `https://apis.map.qq.com` 到 request 合法域名
3. 安装 SDK：
   ```bash
   npm install qqmap-wx-jssdk --legacy-peer-deps
   ```
4. 在 `config/index.ts` 的 `defineConstants` 中配置 Key：
   ```ts
   TENCENT_MAP_KEY: '"你的Key"'
   ```
5. 调用 `Taro.getLocation` 获取用户坐标，再调用腾讯位置服务计算与店铺的直线距离，超过 3km 则禁止下单并提示。

**店铺坐标（松江大学城）：**

```
经度（longitude）：121.2271
纬度（latitude）：30.9956
```

> 注意：`Taro.getLocation` 在微信小程序中需要在 `app.json` 声明 `permission.scope.userLocation`，并在调用前向用户申请授权。

## 8. 余量规则

| 规则项       | 值                        | 说明                     |
| ------------ | ------------------------- | ------------------------ |
| 每日上架时间 | 15:00                     | 用户可开始下单           |

| 每人限购     | 2 杯/单                   | 防止囤积，保证多人可享   |
| 库存初始化   | 每天每种咖啡随机 5～12 杯 | 模拟真实剩余量           |
| 有效期       | 当日 20:00 前核销         | 超过 20:00 订单自动过期  |
| 过期退款     | 自动退款（模拟）          | 未核销订单过期后原路退回 |
